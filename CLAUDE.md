# CLAUDE.md — このリポで作業する際の必須ルール

## 課題管理台帳（BACKLOG.md）の運用 — 必ず従うこと

未実装課題の正準台帳は **リポ直下の `BACKLOG.md`**。以下を強制する:

1. **記録**: 実装アイデア・作問計画・改修案が出たが「今は実装しない」と決まったら、
   その場で `BACKLOG.md` の課題一覧テーブルに1行追記する（ID は `BL-` 連番）。
   ユーザに言われるのを待たず、見送り決定の時点で自発的に記録する。
2. **詳細設計**: 計画が具体化しているもの（トポロジ・要件・採点設計まで固まったもの）は
   `problems/_drafts/<名前>.design.md` に全文を保存し、台帳からリンクする。
   台帳の行は概要1〜2文＋リンクに留める（台帳は一覧性優先）。
3. **着手時**: BACKLOG の課題に着手する時は、まず該当行の状態を更新
   （`未着手` → `PoC中` → `実装中`）してから作業を始める。
4. **完了時**: 完了したら行を「完了アーカイブ」表へ移動し、完了日と成果物
   （問題ID・生成器名など）を備考に書く。design.md は削除しない（履歴として残す）。
5. **重複防止**: 新しい課題を記録する前に既存エントリを確認し、
   既にあるものは行の更新で対応する（新IDを振らない）。

## 補足

- design メモの置き場は `problems/_drafts/`（`memo` はファイルであってディレクトリではない。
  そこには置けない）。
- コミットはユーザの指示があった時のみ。`BACKLOG.md` を更新した際は、
  コミット対象に含めるようユーザに一言添える。
