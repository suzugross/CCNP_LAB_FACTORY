# 問題 ENCOR-IPV6-SLAAC-STATIC-01 : IPv6 自動アドレッシング + 静的経路の発展（難易度4）

## シナリオ
`ENCOR-IPV6-STATIC-01`（手動アドレッシング＋基本3方式）の**発展編**。今回は
**自動アドレッシング（SLAAC / EUI-64）** と、静的経路の発展形（**フローティング / Null0破棄**、＋再帰静的）を扱う。
トポロジは三角形で、RT01—RT03 間に**バックアップの直結リンク**がある。動的ルーティングは使わない。

## トポロジ
```
                         (Lo0)
                    2001:DB8:B::2/128
                         RT02
              Et0/0 ┌─────┴─────┐ Et0/1
   2001:DB8:12::/64 │           │ 2001:DB8:23::/64
       (primary)    │           │  (RA: EUI-64/SLAAC)
              Et0/0 │           │ Et0/0
        RT01 ───────┘           └─────── RT03
   2001:DB8:A::1/128                 2001:DB8:C::3/128
         (Lo0)                            (Lo0)
            └──── 2001:DB8:13::/64 ────┘
            Et0/1   (backup direct)   Et0/1
```

## アドレッシング仕様
3台とも `ipv6 unicast-routing` を有効化。Loopback0 は手動GUA。リンクのアドレッシングは下表のとおり
（**自動アドレッシングの箇所に注目**）。

| ルータ | Loopback0 | Et0/0 | Et0/1 |
|--------|-----------|-------|-------|
| RT01 | `2001:DB8:A::1/128`（手動） | `2001:DB8:12::1/64` + LL `FE80::1` | `2001:DB8:13::1/64` + LL `FE80::1` |
| RT02 | `2001:DB8:B::2/128`（手動） | `2001:DB8:12::2/64` + LL `FE80::2` | **EUI-64**: `2001:DB8:23::/64 eui-64` + LL `FE80::2` |
| RT03 | `2001:DB8:C::3/128`（手動） | **SLAAC**: 自動取得（手動でアドレスを振らない）+ LL `FE80::3` | `2001:DB8:13::3/64` + LL `FE80::3` |

- **RT02 Et0/1**：ホスト部を **EUI-64** で自動生成させる。このIFが RT03 向けに RA（プレフィックス通知）を出す。
- **RT03 Et0/0**：**SLAAC** でアドレスを自動取得する。さらに **RA からデフォルト経路も受け取り**、
  RT03 には**手動のデフォルト経路を書かない**こと（自動取得した `::/0` で上流へ出る）。

## 到達目標（静的経路の発展形）
全ルータが他の全 Loopback に到達できるようにする。**経路の作り方を指定**する。

1. **RT01 → RT02 の Loopback (`2001:DB8:B::2`)**:
   **再帰静的経路**（対向の**グローバルアドレス**をネクストホップに）。
   ※ 出口IFのみを書く「直結指定」は**真のP2Pリンク専用**で、本問のような Ethernet（マルチアクセス）では
   宛先 `/128` をそのリンク上でND解決しようとして失敗する（＝使わない）。
2. **RT01 → RT03 の Loopback (`2001:DB8:C::3`)**: **フローティングスタティック**で冗長化する。
   - **主経路**：RT02 経由（`Et0/0` 側、アドミニストレーティブディスタンスは既定）。
   - **予備経路**：RT01—RT03 の**直結リンク**（`Et0/1` 側）経由で、**AD 200** を付けて待機させる。
   - 平常時は主経路だけが RIB に入り、予備経路は出てこないこと。
3. **RT01**: 自社割当ブロック `2001:DB8:A::/48` のうち**未使用分をブラックホール**する
   **破棄経路（Null0）** を 1 本置く（集約とループ防止の定石）。
4. **RT02**: RT01・RT03 の Loopback へ到達する静的経路を構成（手段は自由）。
5. **RT03**: 上流は SLAAC で得た**デフォルト経路**で到達する（個別の静的経路や手動デフォルトは不要）。

## 制約・ヒント（最小限）
- 動的ルーティングプロトコルは使わない。**静的経路と自動アドレッシングのみ**。
- SLAAC でデフォルト経路まで受け取るには、RA を出す側が**自分をデフォルトルータとして広告**している必要がある。
  受け取る側のコマンドにも「デフォルト経路を入れる」意思表示がいる。
- フローティングの「予備」は、**主経路が生きている間は RIB に現れない**のが正しい状態。
- EUI-64 / SLAAC で付くアドレスのホスト部は機器ごとに決まる（手動で予測しなくてよい）。到達確認は **Loopback** 宛で行う。

## アクセス・採点
SSH `SUZUKI / CCNP`（RT01=.11 / RT02=.12 / RT03=.13）。
```
ansible-playbook playbooks/grade.yml -e problem=ENCOR-IPV6-SLAAC-STATIC-01 --vault-password-file <(printf 'CCNP\n')
```
※ 採点は (1) EUI-64/SLAAC の自動アドレッシング、(2) 各静的経路の**方式**（再帰／フローティング／Null0）と
**フローティングが実際に float しているか**（RIB に主経路のみ）、(3) **実 ping** で全 Loopback 相互到達を能動確認します。
