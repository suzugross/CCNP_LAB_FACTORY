# 問題 ENCOR-PBR-02 : 拠点エッジでの通過トラフィック経路振り分け（難易度4）

## シナリオ
拠点には LAN クライアント **RT04** があり、その配下に 2 つの社内セグメントを持つ。

- **営業部** … `10.1.1.1`（RT04 の Loopback1）
- **開発部** … `10.2.2.1`（RT04 の Loopback2）

RT04 は拠点エッジルータ **RT01** を経由して、データセンター上のサーバ **`3.3.3.3`**（RT03 の Loopback0）と通信する。RT01 からは **WAN-A（RT02 経由）** と **WAN-B（RT03 直結）** の 2 系統があり、OSPF はすでに全台で稼働して全到達できている。

## トポロジ
```
                       WAN-A
            ┌──── RT02 ───────┐
   10.0.12.0/30 e0/0     e0/1 │ 10.0.23.0/30
            │ e0/0            │
  RT04 ─── RT01               RT03 ── (DC: Lo0 3.3.3.3)
  e0/0 e0/2 │ e0/1       e0/0 │
  10.0.14.0/30  └──── WAN-B ──┘
                  10.0.13.0/30
```

| ルータ | Loopback | 直結 |
|--------|----------|------|
| RT04（拠点LAN） | Lo0 `4.4.4.4/32` ／ Lo1 `10.1.1.1/32`(営業) ／ Lo2 `10.2.2.1/32`(開発) | RT01=`10.0.14.1` |
| RT01（拠点エッジ） | Lo0 `1.1.1.1/32` | RT04=`10.0.14.2` ／ RT02=`10.0.12.2` ／ RT03=`10.0.13.2` |
| RT02（WAN-A） | Lo0 `2.2.2.2/32` | RT01=`10.0.12.1` ／ RT03=`10.0.23.2` |
| RT03（WAN-B / DC） | Lo0 `3.3.3.3/32`（DC サーバ） | RT01=`10.0.13.1` ／ RT02=`10.0.23.1` |

- OSPF（プロセス1 / area0）は 4 台とも設定済み・全到達可能（初期状態で疎通あり）。
- 既定では RT04 → `3.3.3.3` は RT01 経由で **WAN-B（RT03 直結）** が選ばれる。

## 到達目標 — RT01 のみ
1. **開発部 (`10.2.2.1`) を送信元**とする DC (`3.3.3.3`) 宛の**通過トラフィック**は **WAN-A（RT02 `10.0.12.2`）経由**で届くようにする。
2. **営業部 (`10.1.1.1`) を送信元**とする DC (`3.3.3.3`) 宛の通信は **現状のまま（最短経路 = WAN-B）** とする。
3. どちらのセグメントからも `3.3.3.3` への到達性は維持すること。

## 制約
- 設定を変更してよいのは **RT01 のみ**。RT02 / RT03 / RT04 は変更しない。
- スタティックルートで OSPF の最短路自体を書き換えてはならない（経路表の最短路は維持）。

## アクセス（SSH, SUZUKI / CCNP）
- RT01:`10.1.10.11` ／ RT02:`10.1.10.12` ／ RT03:`10.1.10.13` ／ RT04:`10.1.10.14`
- ※CML コンソールから直接 RT01 を設定して解いて構いません。

## 採点
```
ansible-playbook playbooks/grade.yml -e problem=ENCOR-PBR-02 --vault-password-file <(printf 'CCNP\n')
```
設定の有無だけでなく、**RT04 から実際に traceroute を流して実際の経路**を検証します
（開発部発＝WAN-A 経由 / 営業部発＝WAN-B 直結のまま）。
