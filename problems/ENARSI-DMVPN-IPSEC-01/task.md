# 問題 : DMVPN 完全版 — Phase 3 + IPsec (IKEv2) で拠点間 VPN を構築する

## シナリオ
本社 (RT01) と 2 つの支店 (RT02 / RT03) が、通信事業者の WAN 網 (RT04) に収容されている。
セキュリティ部門が策定した下記の**設定仕様書**（次世代暗号 = NGE 準拠）に従い、
**1 本のマルチポイントトンネルによる DMVPN** を構築し、全トラフィックを暗号化した上で
全拠点の Loopback を動的ルーティングで相互接続すること。

- RT01 = **ハブ (NHS)** / RT02 = **スポーク1** / RT03 = **スポーク2**
- RT04 = **WAN トランジット**（事業者網・**変更禁止**）

## 既設構成（変更しないこと）
- **Underlay (NBMA)**: 各拠点は RT04 へ /30 で接続済み。他拠点 NBMA へは RT04 経由
  （デフォルトルート設定済み）。RT01=`10.0.14.1/30` / RT02=`10.0.24.1/30` /
  RT03=`10.0.34.1/30`。
- **Loopback0（拠点内ネットワーク相当）**: RT01=`1.1.1.1/32` / RT02=`2.2.2.2/32` /
  RT03=`3.3.3.3/32`。

## 設定仕様書（この値のとおり実装すること）

### 1. DMVPN トンネル
| 項目 | 指定値 |
|------|--------|
| トンネル IF | 全拠点 **`Tunnel0`** の 1 本のみ（マルチポイント GRE） |
| オーバーレイ | **`10.255.0.0/24`**（ハブ `.1` / スポーク1 `.2` / スポーク2 `.3`） |
| トンネル source | 各拠点の WAN 側物理IF |
| GRE トンネルキー | **100** |
| NHRP network-id | **1** |
| NHRP 認証キー | **`DMVPNKEY`** |
| 動作フェーズ | **Phase 3**（定常時の経路はハブ向きのまま・スポーク間トラフィック発生時はハブの誘導でスポーク同士が直接通信する） |
| IF の IPv4 MTU | **1400** バイト |
| TCP MSS 書き換え | **1360** バイト |

### 2. IKE（IKE SA）
| 項目 | 指定値 |
|------|--------|
| バージョン | **IKEv2**（スマートデフォルトに頼らず明示定義すること） |
| 暗号化 | **AES-GCM-256** |
| PRF | **SHA-384** |
| DH グループ | **19** |
| 認証方式 | 事前共有鍵（**双方向**とも） |
| 事前共有鍵 | 全拠点共通 1 本: **`Ss2026#Dmvpn`** |
| デッドピア検出 | **間隔 30 秒 / リトライ 5 秒 / on-demand 方式** |
| SA ライフタイム | 既定値のまま |

### 3. IPsec（Child SA）
| 項目 | 指定値 |
|------|--------|
| ESP 暗号化 | **AES-GCM-256** |
| ESP 整合性 | 追加指定なし（暗号化方式の特性を踏まえて構成すること） |
| カプセル化モード | **transport** |
| PFS | **DH グループ 19** |
| SA ライフタイム | 既定値のまま |

### 4. ルーティング
| 項目 | 指定値 |
|------|--------|
| プロトコル | **EIGRP AS 100** |
| 広告対象 | トンネル区間・各拠点の Loopback0 |

## 到達目標
- ハブで両スポークの DMVPN 登録と 2 本の IKEv2 セッションが確立し、仕様書の
  パラメータ（暗号スイート・カプセル化モード含む）が実機の状態として確認できること。
- 全拠点の Loopback0 が EIGRP で相互到達できること。
- **★Phase 3 の動作**: 定常状態では対向スポーク経路のネクストホップはハブ
  (`10.255.0.1`) のままであること。それでも、スポーク間トラフィックが発生すると
  **スポーク同士が直接、かつ暗号化されたトンネルで**通信すること
  （`show dmvpn` の動的エントリと、スポーク間の IKEv2/IPsec セッションで確認される）。

## 制約
- 既設の underlay / アドレッシング / RT04 は変更しない。
- トンネルは `Tunnel0` の 1 本のみ（スポーク間専用トンネルの追加は禁止）。
- スタティックルートの追加は禁止（既設のデフォルトルートのみ）。
- プロポーザル名・キーリング名・プロファイル名などの**オブジェクト名は任意**。
