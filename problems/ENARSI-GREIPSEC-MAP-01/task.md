# 問題 : GRE over IPsec の構築（crypto map 方式・ハブ+2支店）

## シナリオ
本社 (RT01) と 2 つの支店 (RT02 / RT03) が、通信事業者の WAN 網 (RT04) を介して
接続されている。買収した子会社から引き継いだ VPN 装置群はレガシーな
**crypto map 方式**で運用されており、本社側もこれに合わせる必要がある。
情報システム部が作成した下記の**設定仕様書**に従い、本社–各支店間に
**GRE over IPsec** の VPN を構築し、全拠点の LAN を動的ルーティングで相互接続すること。

- RT01 = **本社 (HQ・ハブ)** / RT02 = **支店1 (Branch1)** / RT03 = **支店2 (Branch2)**
- RT04 = **WAN トランジット**（事業者網・**変更禁止**）
- 支店同士の通信は**本社経由**とする（支店間の直接トンネルは作らない）

## 既設構成（変更しないこと）
- **Underlay (WAN)**: 各拠点は RT04 へ /30 で接続済み。他拠点へは RT04 経由
  （デフォルトルート設定済み）。
  RT01=`10.0.14.1/30` / RT02=`10.0.24.1/30` / RT03=`10.0.34.1/30`。
- **Loopback0**: RT01=`1.1.1.1/32` / RT02=`2.2.2.2/32` / RT03=`3.3.3.3/32`。
- **Loopback10（拠点LAN相当）**: RT01=`192.168.1.1/24` / RT02=`192.168.2.1/24` /
  RT03=`192.168.3.1/24`。

## 設定仕様書（この値のとおり実装すること）

### 1. トンネル方式
| 項目 | 指定値 |
|------|--------|
| 方式 | **GRE トンネル**＋物理IFでの IPsec 暗号化（crypto map 方式） |
| 暗号化の適用箇所 | **各拠点の WAN 側物理IFに crypto map を適用**。トンネルIFへの IPsec プロファイル適用（`tunnel protection`）は**使用禁止** |
| トンネルIF | 本社: 支店1向け **`Tunnel1`** / 支店2向け **`Tunnel2`**。各支店: **`Tunnel1`** |
| トンネルアドレス | 本社–支店1: **`10.255.12.0/30`**（本社 `.1` / 支店 `.2`）<br>本社–支店2: **`10.255.13.0/30`**（本社 `.1` / 支店 `.2`） |
| トンネル source | 各拠点の WAN 側物理IF |
| IF の IPv4 MTU | **1400** バイト |
| TCP MSS 書き換え | **1360** バイト |

### 2. 暗号化対象トラフィック
| 項目 | 指定値 |
|------|--------|
| 対象 | 自拠点 WAN アドレスと対向拠点 WAN アドレス間の **GRE のみ**（それ以外の underlay 通信は暗号化しない） |
| 本社側の収容 | 2 支店分を**同一の crypto map**（エントリを分けて）で WAN 物理IFに収容する |

### 3. IKE（フェーズ1）
| 項目 | 指定値 |
|------|--------|
| バージョン | **IKEv1** |
| 認証方式 | 事前共有鍵（**対向グローバルアドレス限定**で定義） |
| 事前共有鍵の値 | 本社–支店1: **`Cm2026#HqBr1`** / 本社–支店2: **`Cm2026#HqBr2`**（本社では**支店ごとに別鍵**で定義する） |
| 暗号化 | **AES-256** |
| ハッシュ | **SHA-256** |
| DH グループ | **14** |
| SA ライフタイム | 既定値のまま |
| デッドピア検出 | **間隔 10 秒 / リトライ 3 秒 / トラフィック要求時のみ送信（on-demand）** |

### 4. IPsec（フェーズ2）
| 項目 | 指定値 |
|------|--------|
| ESP 暗号化 | **AES-256** |
| ESP 整合性 | **HMAC-SHA-256** |
| カプセル化モード | **transport** |
| PFS | **DH グループ 14** |
| SA ライフタイム | 既定値のまま |

### 5. ルーティング
| 項目 | 指定値 |
|------|--------|
| プロトコル | **OSPF**（プロセスID 任意）**エリア 0** |
| 広告対象 | トンネル区間・各拠点 LAN (`192.168.x.0/24`) のみ（underlay は広告しない） |
| router-id | 各ルータの Loopback0 アドレス |
| 経路要件 | 各拠点は他拠点の LAN を **/24 のプレフィックス長のまま**学習すること。支店間経路は本社のトンネル経由となること |

## 到達目標
- 本社と各支店の間で IPsec セッションが確立し、仕様書のパラメータが実機の状態と
  して確認できること。
- 全拠点 LAN 間（支店1 ⇔ 支店2 を含む）の通信が VPN 経由で成立すること。

## 制約
- 既設の underlay・アドレッシング・RT04 は変更しない。
- crypto map 名・トランスフォームセット名・ACL 名(番号) などの**オブジェクト名は
  任意**（仕様書に無い名前は自由に決めてよい）。
- スタティックルートの追加は禁止（既設のデフォルトルートのみ）。
