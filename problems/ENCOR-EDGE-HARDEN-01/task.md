# 問題 ENCOR-EDGE-HARDEN-01 : エッジ装置ハードニング (AAA + CoPP + インフラACL + 認証NTP)

## シナリオ
エッジルータ **RT01** を本番投入する前に、セキュリティベースラインに沿って **多層でハードニング**します。
隣接ルータ **RT02**（OSPF ピア兼 NTP マスター）は変更できません。

## トポロジ
```
   RT01 ───(OSPF area0, 10.0.12.0/30)─── RT02 (隣接 / NTPマスター, 変更不可)
   1.1.1.1/32                            2.2.2.2/32
```

| ルータ | 役割 | 管理IP(SSH) |
|--------|------|-------------|
| RT01 | ハードニング対象 | `10.1.10.11` |
| RT02 | 隣接ルータ / NTPマスター（**変更不可**） | `10.1.10.12` |

- RT01–RT02 は OSPF area0 で隣接済み。
- RT02 は NTP マスター（認証鍵 `key 1`／文字列はインフラ標準の `CCNP-NTP`、MD5）。

## 到達目標（すべて RT01）
1. **AAA を有効化**する。
   - `new-model` を有効にし、ログイン認証をローカルユーザで行うこと。
2. **CoPP で制御プレーンを保護**する。
   - 制御プレーン宛 **ICMP は帯域制限（ポリシング）**、**Telnet は遮断**すること。
3. **インフラ ACL** で自装置を守る。
   - 隣接セグメント側から **自装置宛の管理アクセス(SSH/Telnet)を遮断**し、それ以外は許可する ACL を、
     隣接インタフェースに適用すること。
4. **認証付き NTP** で RT02 に時刻同期する。
   - RT02 を NTP サーバに指定し、**MD5 認証（key 1 / `CCNP-NTP`）**を有効にすること。

## 制約
- RT02 は変更不可。
- 管理用 SSH（`10.1.10.11`）を切断しないこと。
- 既存の OSPF 隣接を維持すること。

## アクセス
SSH `SUZUKI / CCNP`（例: `ssh SUZUKI@10.1.10.11`）

ansible-playbook playbooks/grade.yml -e problem=ENCOR-EDGE-HARDEN-01 --vault-password-file <(printf 'CCNP\n')
