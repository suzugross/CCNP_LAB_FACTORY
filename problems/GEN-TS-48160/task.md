# 障害対応 GEN-TS-48160 : OSPF 到達性（5 台バックボーン）

## 状況
OSPF（プロセス 1 / エリア 0）で構成された 5 台のルータ網。ある変更作業の後から到達性の不具合が報告されている。

## 受付チケット
> 「**RT05** から **RT01** の Loopback (`2.2.2.2/32`) に到達できない」という申告がありました。
>
> 切り分けて原因を特定し、恒久的に復旧してください。なお他にも影響が出ている可能性があります。

## 構成台帳（Loopback）
- RT01: `2.2.2.2/32`
- RT02: `92.92.92.92/32`
- RT03: `74.74.74.74/32`
- RT04: `31.31.31.31/32`
- RT05: `63.63.63.63/32`

※ 区間アドレスや原因の場所・種類・件数は記載していない。各装置にログインし、`show ip ospf neighbor` / `show ip route` / `show ip ospf interface` 等で実機の状態から切り分けること。

## 完了条件
すべてのルータが、他の全ルータの Loopback を OSPF 経路として学習している状態。

ansible-playbook playbooks/grade.yml -e problem=GEN-TS-48160 --vault-password-file <(printf 'CCNP\n')
