# 問題 ENCOR-OSPF-NSSA-01 : 支店の外部経路を HQ へ運びつつ支店の RIB を絞る

## シナリオ
HQ(area 0) と支店(area 1) を ABR(RT02) で接続した OSPF 網がある。
支店ルータ RT03 は、支店ローカルの外部ネットワーク `198.51.100.0/24` を
static で保有している（インターネット側 ISP から払い出された区画、という想定）。

運用チームからの要望は次の 2 つ:

1. **この外部ネット `198.51.100.0/24` を HQ 全体へ配ってほしい。**
   HQ の Core ルータ RT01 が、この区画を OSPF の外部経路として学習できること。
2. **同時に、支店ルータ RT03 の RIB は太らせたくない。**
   RT03 が area 1 の外（HQ 側）を見るために持つ OSPF 経路は、
   デフォルトルート 1 本だけにすること。

## トポロジ
```
[Lo0=1.1.1.1]        [Lo0=2.2.2.2]         [Lo0=3.3.3.3]
RT01 (HQ Core) ──── RT02 (ABR) ────────── RT03 (支店)
        10.0.12.0/30        10.0.23.0/30        + 外部 198.51.100.0/24 (static)
         area 0               area 1
```

| ルータ | 役割 | Lo0 | 管理IP |
|--------|------|-----|--------|
| RT01 | HQ Core（area 0 内部・**変更不可**） | 1.1.1.1 | `10.1.10.11` |
| RT02 | ABR（area 0 / area 1 境界・変更可） | 2.2.2.2 | `10.1.10.12` |
| RT03 | 支店（area 1 内部・外部区画を保有・変更可） | 3.3.3.3 | `10.1.10.13` |

## 初期状態（確認用）
- RT03 は `ip route 198.51.100.0 255.255.255.0 Null0` を持つが、まだ OSPF には出していない。
- area 1 は現在「通常エリア」。RT03 の RIB には HQ 側の個別経路（`O IA 1.1.1.1/32` 等）が見えている。
- RT01 はまだ `198.51.100.0/24` を学習していない。

## 到達目標
1. **RT01(HQ Core) が `198.51.100.0/24` を OSPF の外部経路として学習している**こと。
2. **RT03 が area 1 の外を見る OSPF 経路は、デフォルトルート 1 本だけ**であること。
   （HQ 側の個別 inter-area 経路 `1.1.1.1/32` `2.2.2.2/32` は RT03 の RIB に存在しないこと）
3. RT02 ↔ RT03 の OSPF 隣接 (Full) を維持すること。

## 制約
- **変更してよいのは RT02 と RT03 のみ**（RT01 は変更不可）。
- area 番号（area 0 / area 1）と OSPF プロセス ID(1) は変更しない。
- 既存のインタフェース IP・network 文は変更しない（設定の **追加** で実現すること）。
- デフォルトルートやスタティックの「見せかけ」手動注入で誤魔化さないこと。

> ヒント控えめ：**どの area タイプを使うか・どちら側に何を書くか**は記載していません。
> 「外部経路をエリアに持ち込みつつ、エリア内の RIB は絞る」という 2 つの要件が同時に
> 成り立つ area の性質は何か、を考えてください。
