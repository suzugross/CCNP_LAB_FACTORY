# UM2-BUILD-02: Untrustワンアーム構成の構築（ワンアームLB版）

あなたはデータセンターの新規区画に、設計書に基づく **Untrustワンアーム構成** を
1から構築します。物理はワンアーム（FW/LB を L3スイッチの腕として収容）、
論理は VRF とタグVLAN で分離するデザインパターンです。

★本問は UM2-BUILD-01 の姉妹問です。**LB も FW と同じ「腕1本のワンアーム」**で
収容します（上流/下流を 1 物理IFに dot1q で多重）。

## トポロジ

```
                BACKBONE(据付)  Lo0=198.51.100.1
                     |
                  [BB-SW]
                 /        \           VLAN10 (アクセス)
             L3SW1 ------ L3SW2       ← Gi0/3同士: トランク
             /  |          |  \  \
          FW1  LB1        FW2 LB2 DMZ-SV(据付)
              (腕1本)        (腕1本)   ← DMZ-SV = L3SW2 Gi1/0
   FW HA: FW1 Gi0/1—L3SW1 Gi0/2 / FW2 Gi0/1—L3SW2 Gi0/2 (VLAN245アクセス)
   USER-PC(据付)=L3SW1 Gi1/0
```

| リンク | 種別 |
|---|---|
| L3SW Gi0/0 ↔ BB-SW | アクセス VLAN10 |
| L3SW1 Gi0/1 ↔ FW1 Gi0/0（L3SW2↔FW2 同） | トランク 243,244,254 |
| L3SW1 Gi0/2 ↔ FW1 Gi0/1（L3SW2↔FW2 同） | アクセス VLAN245（FW HA） |
| L3SW1 Gi0/3 ↔ L3SW2 Gi0/3 | トランク 100,243,244,245,**251**,254 |
| L3SW1 Gi1/1 ↔ LB1 Gi0/0（L3SW2↔LB2 同） | **トランク 251,254（LBの腕・上下を多重）** |
| L3SW2 Gi1/0 ↔ DMZ-SV | アクセス VLAN251（サーバ収容・ポート設定は受講者） |
| L3SW1 Gi1/0 ↔ USER-PC | アクセス VLAN100 |

## 要件

- **R1**: L3SW×2 に UNTRUST / TRUST の 2 VRF を作成し、上表どおり SVI を収容すること。
  **VLAN 245 / 251 / 254 に SVI を作らないこと**（ゾーン分離の要。DMZ 系セグメントの
  L3 は FW / LB 側だけが持つ）
- **R2**: 全 FHRP は 1号機 Active（priority 110 / preempt）・2号機 100 とする
- **R3**: L3SW1 のアップリンク（Gi0/0）障害時、**Untrust VRF の VIP（Vlan243）だけ**が
  2号機へ切り替わるよう、インターフェーストラッキング（decrement 30）を設定すること
- **R4**: FW は Active/Standby フェールオーバー構成とし、**セカンダリ(FW2)には
  failover 関連の最小設定のみ**を投入して残りは同期させること。
  **データリンク障害時にも自動で切り替わること**
- **R5**: FW のゾーンは outside(0) / inside(100) / dmz(50)。ワンアーム 1 物理IFに
  dot1q サブIFで収容すること
- **R6**: outside から許可する着信は「VIP宛の ICMP と HTTP」のみとし、
  ユーザーVLAN発の外向き通信は 2.2.2.10 へ PAT すること
- **R7**: LB×2 は**ワンアーム**とする: **Gi0/0 の 1 本に dot1q サブIF 254（上流）と
  251（下流）を多重**する。VIP 172.16.250.1 を NAT で終端し、サーバの GW
  （172.16.251.1）を提供すること。
  ★この形では LB の腕障害に対する FHRP の追加トラッキングは**不要**です —
  なぜ不要で済むのか、構成を組みながら考察してください（採点後に解説します）
- **R8**: ICMP がゾーンをステートフルに通過できること（FW の inspection）

## 完成条件（この状態で採点します）

1. USER-PC から `ping 198.51.100.1` が成功（送信元は 2.2.2.10 に PAT）
2. BACKBONE から `ping 2.2.2.1` が成功（FW→LB の二段 NAT でサーバまで到達）
3. USER-PC から `wget http://172.16.250.1/`（VIP 経由）が成功
4. FHRP / FW フェールオーバー / トラッキングが上記仕様どおりの状態表示
5. ★DMZ サーバへは **VIP 経由でアクセスできれば正**（実IP 直宛の通信は本設計では
   成立しません）

## ログイン・注意

- IOS（L3SW/LB）: コンソール、`SUZUKI` / `CCNPccnp`（enable 同）
- ASA（FW1/FW2）: 工場出荷状態から構築します。初回 `enable` で新パスワード設定を
  求められるので **`CCNPccnp`** を設定（8文字以上必須）。hostname 等も仕様に沿って設定
- 据付機器（BACKBONE / USER-PC / DMZ-SV）は**変更禁止**（設定済み）。
