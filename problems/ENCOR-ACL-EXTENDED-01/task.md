# 問題 ENCOR-ACL-EXTENDED-01 : 拡張 ACL でサーバ宛の Telnet と ICMP を遮断

## シナリオ
RT03 上のサーバ (`3.3.3.3`) を保護するため、**RT01 側から来るトラフィック**のうち
**このサーバ宛の Telnet と ping(ICMP) だけを遮断**したい。それ以外の通信
（業務トラフィックやルーティングプロトコル）は一切止めないこと。

フィルタは **RT02 の RT01 向けインタフェース (`Ethernet0/0`) に inbound** で適用します。

## トポロジ
```
RT01 ──(OSPF area0)── RT02 ──(OSPF area0)── RT03 [server 3.3.3.3]
     10.0.12.0/30          10.0.23.0/30
```
| ルータ | Loopback0 | 直結 |
|--------|-----------|------|
| RT01 | 1.1.1.1/32 | RT02=10.0.12.2 |
| RT02 | 2.2.2.2/32 | RT01=10.0.12.1 / RT03=10.0.23.1 |
| RT03 | 3.3.3.3/32 (server) | RT02=10.0.23.2 |

- OSPF (プロセス1 / area0) は全ルータ設定済み・全到達可能（初期状態で疎通あり）。

## 到達目標 — RT02 のみ
1. 拡張 ACL（名前付き・名前は任意）を作り、**RT02 の `Ethernet0/0` に inbound** で適用する。
2. ACL の効果:
   - **サーバ `3.3.3.3` 宛の Telnet (TCP 23) を遮断**する。
   - **サーバ `3.3.3.3` 宛の ICMP を遮断**する。
   - **上記以外はすべて許可**する（業務通信・OSPF などを止めない）。

## 制約
- 設定するのは **RT02 のみ**。RT01 / RT03 は変更しない。
- ★ ACL の末尾の暗黙の deny に注意。必要な通信（OSPF 含む）まで巻き込むと
  ルーティングが壊れて減点になる。

## アクセス（SSH, SUZUKI / CCNP）
- RT01: `10.1.10.11` / RT02: `10.1.10.12` / RT03: `10.1.10.13`
