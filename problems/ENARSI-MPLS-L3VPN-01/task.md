# 問題 ENARSI-MPLS-L3VPN-01 : MPLS L3VPN 基礎構築（マルチカスタマー）（難易度3）

## シナリオ
あなたは通信事業者（SP）のネットワークエンジニアです。PE 2台と P 1台から成る MPLS コアを
新規に立ち上げ、顧客2社（**CUST_A** / **CUST_B**）へサイト間 L3VPN サービスを提供します。

> **重要:** 両顧客は社内で**同じプライベートアドレス帯（172.16.1.0/24, 172.16.2.0/24）を
> 使用しています**。それぞれの VPN 内では両サイトが疎通し、顧客間は決して到達・混線
> しないこと。これが本サービスの契約条件です。

```
 CUST_A CE1                                          CUST_A CE2
   RT04 ---\                                        /--- RT05
            RT01 ========= RT02 ========= RT03
   RT06 ---/ (PE1)          (P)          (PE2) \--- RT07
 CUST_B CE1                                          CUST_B CE2
            └───────────── SPコア ────────────┘
```

## 構成（初期状態で投入済み）

### コア（RT01/RT02/RT03 = あなたの構成対象）
| 項目 | 値 |
|------|-----|
| RT01(PE1) Loopback0 | 1.1.1.1/32 |
| RT02(P)   Loopback0 | 2.2.2.2/32 |
| RT03(PE2) Loopback0 | 3.3.3.3/32 |
| RT01–RT02 | 10.1.12.0/30（RT01=.1 / RT02=.2）設定済み |
| RT02–RT03 | 10.1.23.0/30（RT02=.1 / RT03=.2）設定済み |

ルーティング・MPLS・BGP・VRF は**一切未設定**。PE の CE 向けインタフェースは
**IP 設定済みだが、まだ VRF に収容されていない**（グローバルテーブルのまま）。
仕様 4 の通りに VRF へ収容すること。

### CE（RT04〜RT07 = 顧客管理機器・設定済み・変更禁止）
各 CE はサイト LAN（Loopback0）と PE 向けデフォルトルートを持つ。

| CE | 顧客/サイト | PE との接続 | サイト LAN (Lo0) |
|----|------------|-------------|------------------|
| RT04 | CUST_A site1 | 192.168.1.0/30（CE=.1 / PE1=.2） | 172.16.1.1/24 |
| RT05 | CUST_A site2 | 192.168.2.0/30（CE=.1 / PE2=.2） | 172.16.2.1/24 |
| RT06 | CUST_B site1 | 192.168.11.0/30（CE=.1 / PE1=.2） | **172.16.1.9/24** |
| RT07 | CUST_B site2 | 192.168.12.0/30（CE=.1 / PE2=.2） | **172.16.2.9/24** |

## 設計仕様（この通りに構成すること）

1. **コア IGP** — OSPF プロセス 1 / area 0。対象はコアリンクと Loopback0 のみ。
   CE 向けインタフェース・顧客経路を IGP に入れないこと。
2. **MPLS** — コアリンクで LDP を有効化。LDP ルータ ID は Loopback0 に固定（force）。
   CE 向けインタフェースでは有効化しないこと。
3. **MP-BGP** — AS **65000**。PE1–PE2 の **Loopback0 間** iBGP を 1 本張り、
   **VPNv4 アドレスファミリのみ**有効化する（AF 方式。ipv4 unicast の既定有効化は無効に
   すること）。**P(RT02) には BGP を設定しないこと。**
4. **VRF**（両 PE で共通の値を使うこと）

   | VRF | RD | RT (import / export) | 収容インタフェースと PE 側 IP |
   |-----|----|---------------------|------------------------------|
   | CUST_A | 65000:100 | 65000:100 / 65000:100 | PE1: RT04 向け 192.168.1.2/30 ・ PE2: RT05 向け 192.168.2.2/30 |
   | CUST_B | 65000:200 | 65000:200 / 65000:200 | PE1: RT06 向け 192.168.11.2/30 ・ PE2: RT07 向け 192.168.12.2/30 |

5. **PE-CE ルーティング** — スタティック。各 PE で、直収サイトの LAN（/24）への経路を
   当該 VRF のルーティングテーブルに CE 宛てで設定する。
6. **再配布** — 各 VRF の connected / static を VPNv4 へ再配布し、対向 PE のサイトへ届ける。

## 到達目標（最終状態）
- **CUST_A**: RT04 から `ping 172.16.2.1 source 172.16.1.1` が成功（サイト間疎通）。
- **CUST_B**: RT06 から `ping 172.16.2.9 source 172.16.1.9` が成功。
- **顧客間分離**: CUST_A のサイトから CUST_B のアドレス（.9）へは到達できないこと。
- コア転送は**ラベルスイッチング**で行われること
  （PE1 で `traceroute 3.3.3.3 source 1.1.1.1` するとラベルが観測できる状態）。

## 制約
- CE（RT04〜RT07）の設定変更禁止。
- スタティックルートは仕様 5 の PE-CE 用途以外で使用しないこと。
- 管理用 VRF `MGMT` と Ethernet0/3 には触れないこと。
