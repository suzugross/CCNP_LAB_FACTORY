# 問題 DMVPN-POC-01 : DMVPN マルチスポークで拠点間を直接接続する (Phase 2)

## シナリオ
本社 (RT01) と 2 つの支店 (RT02 / RT03) が、共通の WAN (RT04) に収容されている。
ハブ&スポーク型の **DMVPN** を 1 本のマルチポイントトンネルで構築し、拠点の内部
プレフィックスを動的ルーティングで相互に学習させたい。さらに、**支店間の通信は
本社 (ハブ) を経由せず、支店どうしが直接**やり取りできるようにすること。

- RT01 = **ハブ (NHS)**
- RT02 = **スポーク1** / RT03 = **スポーク2**
- RT04 = **WAN トランジット**（NBMA 中継・**変更禁止**）

## 既設構成（変更しないこと）
- **Underlay (NBMA)**: 各エッジは RT04 へ /30 で接続し、他拠点の NBMA へは
  RT04 経由（デフォルトルート設定済み）。
  - RT01 = `10.0.14.1` / RT02 = `10.0.24.1` / RT03 = `10.0.34.1`
  - RT04 = `10.0.14.2` / `10.0.24.2` / `10.0.34.2`
- **Loopback0**（各拠点の内部プレフィックス・到達対象）:
  - RT01 = `1.1.1.1/32` / RT02 = `2.2.2.2/32` / RT03 = `3.3.3.3/32`

## 到達目標
- 1 本の **`Tunnel0`** でハブ&スポークの**マルチポイント**オーバーレイを確立する。
- 両スポークがハブに登録され、ハブ⇔各スポークのオーバーレイが **UP** になること。
- オーバーレイ上で **EIGRP AS 100** を動かし、全拠点が互いの `Loopback0` を学習する。
- **★支店間 (RT02 ⇔ RT03) の通信はハブを経由せず直接行われること。**
  すなわち、各スポークが学習する「対向スポークの内部プレフィックス」の
  ネクストホップは、**ハブではなく対向スポーク自身**であること。

## 指定値（採点が固定値で判定するため必ず従うこと）
- トンネルインタフェース: **`Tunnel0`**
- オーバーレイ IP 空間: **`10.255.0.0/24`**
  （ハブ = `.1` / スポーク1 = `.2` / スポーク2 = `.3`）
- NHRP **network-id = `1`** / NHRP **認証キー = `DMVPNKEY`**
- ルーティングプロトコル: **EIGRP AS `100`**
- ハブの NBMA アドレス = `10.0.14.1`

## 制約
- 既設の underlay / アドレッシング / RT04 は変更しない。
- 暗号化 (IPsec) は**不要**。
- トンネルは 1 本（`Tunnel0`）のみで構成すること（追加トンネルは作らない）。
