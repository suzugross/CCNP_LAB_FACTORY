# 問題 ENARSI-MPLS-L3VPN-03 : MPLS L3VPN 応用2 — バックドアリンクと経路選択（難易度5）

## シナリオ
あなたは通信事業者（SP）のネットワークエンジニアです。ENARSI-MPLS-L3VPN-02 で構築した
L3VPN サービス（CUST_A / CUST_B）は**稼働中**で、PE-CE は OSPF、コアは MP-BGP VPNv4 で
正常に動いています。

顧客 **CUST_A** から**障害チケット**が届きました。

> **チケット — サイト間の大容量転送が遅い / タイムアウトする**
> CUST_A の site1 と site2 の間に、コスト削減のため**両サイトを直結する低速の予備回線
> （バックドア）**を最近開通させた。これは**あくまで予備**で、通常時はこれまで通り
> **SP の MPLS コアを主経路**として使う契約のはずだ。ところが開通後、**サイト間の通信が
> ことごとくこの低速回線に流れてしまい**、大容量転送が遅い・不安定になっている。
>
> **要望:** 通常時は **MPLS コアを主経路**とし、**バックドアはあくまで予備**
> （MPLS コア障害時のみ経路が切り替わる）となるように、SP 側（PE）で是正してほしい。
> バックドア回線自体は**予備として残す**こと（撤去しないこと）。

```
 CUST_A site1                                   CUST_A site2
   RT04(CE) ─ e0/0 ─┐               ┌─ e0/0 ─ RT05(CE)
      │ e0/1       RT01 ═ RT02 ═ RT03        e0/1 │
      └──────── バックドア (area0・低速) ───────────┘
   RT06(CE) ── CUST_B site1     CUST_B site2 ── RT07(CE)   ← 対照(バックドア無)
            └──────── SPコア (MPLS L3VPN) ─────────┘
```

## 構成（初期状態で投入済み）

- **コア＋L3VPN は稼働中**（OSPF・LDP・MP-BGP VPNv4・VRF 収容・PE-CE OSPF・相互再配布・
  広告制御・MSS すべて構成済み。02 の完成形）。
- **CUST_A の site1(RT04)↔site2(RT05) に直結バックドア**（172.16.9.0/30, area 0,
  **OSPF cost 500 の低速回線**）が開通済み。**現在サイト間はこのバックドア経由**になっている。
- **CUST_B にはバックドアは無い**（常に MPLS コア経由・対照）。

### CE（RT04〜RT07 = 顧客管理機器・変更禁止）
| CE | 顧客/サイト | router-id | サイト LAN | 機器管理 | バックドア |
|----|------------|-----------|-----------|----------|-----------|
| RT04 | CUST_A site1 | 4.4.4.4 | 172.16.1.1/24 | 10.99.4.1/32 | 172.16.9.1/30 (cost 500) |
| RT05 | CUST_A site2 | 5.5.5.5 | 172.16.2.1/24 | 10.99.5.1/32 | 172.16.9.2/30 (cost 500) |
| RT06 | CUST_B site1 | 6.6.6.6 | 172.16.1.9/24 | 10.99.6.1/32 | — |
| RT07 | CUST_B site2 | 7.7.7.7 | 172.16.2.9/24 | 10.99.7.1/32 | — |

## 是正の要件（この状態にすること）

1. **通常時、CUST_A の site1↔site2 は MPLS コア（PE 経由）を主経路**とする。
   RT04 の対向サイト LAN(172.16.2.0/24) への経路が **PE(RT01) 向け**になっていること。
2. **バックドアは予備**として残す（インタフェース・OSPF 隣接は生かしたまま）。
   MPLS コア障害時には自動的にバックドア経由へ切り替わること。
3. **既存のループ防止（PE がコアの VPN 経路を BGP へ再注入し返さない仕組み）を壊さない**こと。
4. **CUST_B（対照）は現状どおり MPLS コア経由のまま**であること。
5. 顧客間分離・広告制御（10.99 が対向へ出ない）は 02 と同様に維持すること。

## 診断のヒント
- 「コストを下げれば直る」とは限らない。**なぜ低速(cost 500)のバックドアが MPLS コアより
  優先されているのか**——RT04 で対向 LAN 経路の**OSPF 経路種別（intra / inter）**を確認せよ。

## 制約
- **操作してよいのは PE（RT01 / RT03）のみ**。CE（RT04〜RT07）とバックドア回線は変更禁止。
- P（RT02）は引き続き BGP を持たないこと。
- コアの既存構成（OSPF 1 / LDP / BGP セッション / VRF 定義・PE-CE OSPF・再配布）を壊さないこと。
- スタティックルートは使用しないこと。
- 管理用 VRF `MGMT` と Ethernet0/3 には触れないこと。
