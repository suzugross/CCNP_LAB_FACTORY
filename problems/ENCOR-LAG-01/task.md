# 問題 ENCOR-LAG-01 : スイッチ間リンクアグリゲーション

## シナリオ
スイッチ SW01 と SW02 は 2 本の物理リンクで接続されています。この 2 本を **1 本の論理リンクに束ねて**、
帯域と冗長性を確保したいと考えています。

## 構成（初期状態で投入済み）
- SW01–SW02 間に物理リンクが 2 本（`Ethernet0/0` と `Ethernet0/1`、いずれも up 済み・未束ね）。
- 管理 IP（採点で使用）: SW01 = 10.1.10.11 / SW02 = 10.1.10.12（telnet 可）。

## 到達目標
1. SW01–SW02 間の 2 本の物理リンクを **1 つの論理リンクに束ねる**。
2. **2 本の物理リンクが両方ともアクティブに束へ参加**していること（両方が束の一員として up）。
3. 束ねは **両端の機器が互いにネゴシエーションして動的に確立する方式**とすること
   （片側任せの静的固定ではなく、対向と折衝して束を組むこと）。

## 制約
- 2 本のリンクは必ず同一の束に入れること（バラバラにしない）。
- 管理用インタフェース／管理 VLAN には触れないこと。

## ログイン（telnet）/ 採点
```
! telnet 10.1.10.11      # SW01 （user SUZUKI / pass CCNP）
! telnet 10.1.10.12      # SW02

ansible-playbook playbooks/grade.yml -e problem=ENCOR-LAG-01 --vault-password-file <(printf 'CCNP\n')
```
> ※ このスイッチ(IOL L2)は SSH 鍵生成に非対応のため telnet でログインします。採点も telnet で状態を収集します。
