# 問題 ENARSI-OSPF-MADJ-01 : ABR 間の直結リンクを活かせ

難易度: ★★★★☆（4/5）

## シナリオ

ある企業ネットワークは、西サイト（area 1）と東サイト（area 2）を
バックボーン（area 0）で接続している。バックボーンは RT03/RT04 を経由する
**低速な長距離回線**（各区間 OSPF cost 100）で構成されている。

一方、ABR である RT01–RT02 間には**高速な直結リンク**が存在するが、
このリンクは監視チーム所有の **NMS セグメント（area 100）** に収容されており、
監視プローブ（172.31.100.1 / 172.31.100.2）の疎通に使われている。

利用部門から「西サイト（RT05）と東サイト（RT06）の間の通信が遅い」と
クレームが入った。調査すると、RT05↔RT06 のトラフィックは高速な直結リンクを
使わず、低速なバックボーン経由で 5 ホップも迂回していることが分かった。

## トポロジ

```
             area 1              area 0 (cost 100 ×3)             area 2
  RT05 ─────── RT01 ────── RT03 ────────── RT04 ────── RT02 ─────── RT06
 Lo0=5.5.5.5    │  10.1.13.0/30   10.1.34.0/30  10.1.24.0/30 │    Lo0=6.6.6.6
  10.1.15.0/30  │                                            │  10.1.26.0/30
                └──────────── 10.1.12.0/30 ──────────────────┘
                        area 100 (NMS・高速直結)
```

- 各リンクの /30 は小さいルータ番号側が .1（例: RT01–RT02 は RT01=.1 / RT02=.2）
- Lo0: RTxx = x.x.x.x/32（RT05=5.5.5.5 area1 / RT06=6.6.6.6 area2）
- NMS プローブ: RT01 Lo100=172.31.100.1/32、RT02 Lo100=172.31.100.2/32（area 100）
- 全ルータで OSPF プロセス 1 が稼働済み・全隣接 UP・全経路到達可能

## 到達目標

1. RT05 (5.5.5.5) ↔ RT06 (6.6.6.6) のトラフィックが、**往復とも**
   RT01–RT02 の直結リンクを通ること（バックボーンの RT03/RT04 を経由しない）。
2. 直結リンク障害時には、トラフィックが自動的に RT03/RT04 経由へ切り替わること
   （追加設定は不要な構成であること）。
3. NMS セグメント（area 100）の既存機能を壊さないこと:
   - RT01–RT02 間の area 100 の隣接関係を維持する
   - 監視プローブ 172.31.100.1 / 172.31.100.2 への到達性を維持する

## 制約

- **virtual-link の使用禁止**（社内標準で不許可）。
- **既存インタフェース・Loopback のエリア割当を変更しないこと**
  （直結リンクを area 100 から外す/移すことは監視チームとの合意上不可）。
- スタティックルート・PBR による「見せかけ」の解決は禁止。
- 物理リンクの増設・IP アドレスの変更は不可。
- 既設の OSPF cost 値（バックボーンの 100）は変更しないこと。

## アクセス

- CML コンソール（SUZUKI / CCNP, enable CCNP）
- SSH の場合は provision 時に割当てられた mgmt IP
  （`topologies/_generated/ENARSI-OSPF-MADJ-01/mgmt_map.yml` 参照）
