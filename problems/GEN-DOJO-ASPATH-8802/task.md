# 問題 GEN-DOJO-ASPATH-8802 : as-path access-list 道場（10本ノック・難易度3）

## この問題について（型稽古）

上流ルータ **RT02 (FEEDER)** はテスト用の経路発生装置で、RT01 との間に
**並列の eBGP セッションを4本**張り、それぞれ別の隣接 AS
（**AS65099 / AS65010 / AS65020 / AS65030**）として振る舞いながら、
多彩な **AS_PATH** を持つ経路群（battery）を広告している。
パス上には中継/起源 AS として **AS65100 / AS65200 / AS65300** が現れる。
また RT01 自身もローカル経路（AS_PATH 空）を2本 BGP に注入している。

あなたは **RT01 (AS65001)** 上で、下記 10 個の小課題それぞれについて
**指定された番号の ip as-path access-list を1本定義する**。それだけの問題である。

- as-path access-list は**定義するだけ**でよい（ネイバーへの適用は不要）。
- 採点は正規表現の書き方ではなく「**どの経路を通し、どの経路を落とすか**」の
  意味的一致で行う（1課題 = 10点・部分点なし）。

```
  RT01 (TARGET, AS65001)  Lo0=1.1.1.1  ＋ローカル経路×2 (AS_PATH 空)
    ||||  ← 並列 eBGP×4 (直結 + loopback間×3)
  RT02 (FEEDER)  ← AS65099/65010/65020/65030 として battery 広告(変更禁止)
```

## 素振り（自己確認）のしかた

- battery 全体と AS_PATH の観察: `show ip bgp`（Path 列を読む）
- 自分のリストの効果を**その場で確認**: `show ip bgp filter-list <番号>`
  （read-only。BGP テーブルへの適用表示であり、セッションに影響しない）
- 正規表現の素振り（リストを定義せず試せる）: `show ip bgp regexp <正規表現>`

## 課題（リスト番号は厳密に一致させること）

### 課題1 — リスト番号 `1`

> **隣接 AS 65020 から受信した**経路（AS_PATH の先頭が 65020）をすべて許可せよ（起源がどの AS かは問わない）。

### 課題2 — リスト番号 `2`

> **隣接 AS 65030 から受信した**経路（AS_PATH の先頭が 65030）をすべて許可せよ（起源がどの AS かは問わない）。

### 課題3 — リスト番号 `3`

> 隣接 AS **65010** から直接受信し、かつ **AS 65010 自身が起源**である経路（AS_PATH に他の AS が一切現れない）のみを許可せよ。

### 課題4 — リスト番号 `4`

> 隣接 AS **65099** から直接受信し、かつ **AS 65099 自身が起源**である経路（AS_PATH に他の AS が一切現れない）のみを許可せよ。

### 課題5 — リスト番号 `5`

> RT01 **自身が生成**した経路（AS_PATH が空の経路）のみを許可せよ。外部から受信した経路は一切許可しないこと。

### 課題6 — リスト番号 `6`

> **AS 65300 が起源**（AS_PATH の末尾が 65300）の経路をすべて許可せよ。経由しているだけの経路は許可しないこと。

### 課題7 — リスト番号 `7`

> AS_PATH のどこかに **AS 65200 を含む**経路（位置を問わない）をすべて許可せよ。それ以外は許可しないこと。

### 課題8 — リスト番号 `8`

> **AS 65200 が起源**（AS_PATH の末尾が 65200）の経路をすべて許可せよ。経由しているだけの経路は許可しないこと。

### 課題9 — リスト番号 `9`

> **AS 65200 を経由するが、AS 65200 が起源ではない**経路のみを許可せよ（トランジットとしてのみ 65200 を通った経路の抽出）。

### 課題10 — リスト番号 `10`

> 隣接 AS **65010** が **AS パスプリペンド**を行って広告している経路（AS_PATH が 65010 の**2回以上**の繰り返しだけで構成される）のみを許可せよ。プリペンド無しで AS 65010 から直接受信している経路は含めないこと。

## 遵守事項

1. **RT02 (FEEDER) の設定変更は禁止**（show による状態確認は可）。
2. RT01 の **BGP・インタフェース・ルーティング設定の変更は禁止**
   （作業は ip as-path access-list の定義のみ）。
3. リスト番号は課題の指定どおり（`1`〜`10`）に一致させること。
