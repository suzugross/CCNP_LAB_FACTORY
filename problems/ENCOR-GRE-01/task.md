# 問題 ENCOR-GRE-01 : GRE トンネルで支店プライベートLANを接続

## シナリオ
HQ (RT01) と Branch (RT03) は、プロバイダ網 (RT02) 経由で接続されている。
プロバイダ網の中は OSPF area 0 が事業者ポリシーで配備済みだが、
お客様のプライベートアドレス空間 (Loopback10) はプロバイダ網に広告できない。
両拠点のプライベートLANが相互に到達できるよう、**オーバーレイ**で接続せよ。

## 既設構成
- **Underlay**: OSPF area 0
  - RT01–RT02: `10.0.12.0/30` (RT01=.1 / RT02=.2)
  - RT02–RT03: `10.0.23.0/30` (RT02=.1 / RT03=.2)
  - Loopback0 (RID / underlay 上で到達可): RT01=`1.1.1.1`, RT02=`2.2.2.2`, RT03=`3.3.3.3`
- **プライベート LAN** (Loopback10, underlay には載せていない):
  - RT01: `10.10.1.1/32`
  - RT03: `10.10.3.1/32`

## 到達目標
- RT01 ↔ RT03 のオーバーレイを確立し、互いの `10.10.x.1/32` を**動的ルーティングで**学習させる。
- 学習に使うのは **EIGRP AS 100**。
- プライベート (Loopback10) は **underlay (OSPF) には広告しないこと**。

## 制約
- **RT02 (プロバイダ機器) は変更不可**。
- 既設の OSPF / アドレッシングは変更しない。
- オーバーレイ用に使うインタフェース番号は **Tunnel100** に統一する。
- オーバーレイの IP 空間は `172.16.13.0/30` を使う (RT01=.1, RT03=.2)。
- EIGRP の AS 番号は **100** を使う。
