# 問題 DMVPN-PHASE3-01 : DMVPN Phase 3 でスポーク間をオンデマンド直結する

## シナリオ
本社 (RT01) と 2 つの支店 (RT02 / RT03) が共通の WAN (RT04) に収容されている。
ハブ&スポーク型の **DMVPN** を 1 本のマルチポイントトンネルで構築する。支店間の
通信は、**普段はハブを向いた経路のまま**にしておき、実際に支店間トラフィックが
発生したときに**オンデマンドで直接トンネルを張る (Phase 3)** 方式にすること。

- RT01 = **ハブ (NHS)** / RT02 = **スポーク1** / RT03 = **スポーク2**
- RT04 = **WAN トランジット**（NBMA 中継・**変更禁止**）

## 既設構成（変更しないこと）
- **Underlay (NBMA)**: 各エッジは RT04 へ /30 接続、他拠点 NBMA へは RT04 経由
  （デフォルトルート設定済み）。RT01=`10.0.14.1` / RT02=`10.0.24.1` / RT03=`10.0.34.1`。
- **Loopback0**: RT01=`1.1.1.1` / RT02=`2.2.2.2` / RT03=`3.3.3.3`。

## 到達目標
- 1 本の **`Tunnel0`** でマルチポイントオーバーレイを確立し、両スポークがハブに登録され
  UP になること。EIGRP AS100 で全拠点が互いの `Loopback0` を学習する。
- **★Phase 3 の動作**:
  - **定常状態では、スポークが学ぶ「対向スポークの内部プレフィックス」の
    ネクストホップは ハブ (`10.255.0.1`)** のままであること
    （ハブはネクストホップを自分に保つ）。
  - それでも、**支店間 (RT02⇔RT03) のトラフィックが発生すると、ハブの誘導により
    スポークどうしが直接トンネルを張り**（`show dmvpn` に対向スポークへの
    動的エントリが UP で現れる）、データはハブを経由せず直接流れること。

## 指定値（採点が固定値で判定するため必ず従うこと）
- トンネル: **`Tunnel0`** / オーバーレイ **`10.255.0.0/24`**（Hub`.1`/Sp1`.2`/Sp2`.3`）
- NHRP **network-id `1`** / **認証キー `DMVPNKEY`**
- ルーティング: **EIGRP AS `100`** / ハブの NBMA = `10.0.14.1`

## 制約
- 既設の underlay / アドレッシング / RT04 は変更しない。
- 暗号化 (IPsec) は不要。トンネルは `Tunnel0` 1 本のみ。
- **ハブはネクストホップをスポークに委譲しない**（＝対向スポーク経路の next-hop は
  ハブのまま）。スポーク間の直結は**経路制御ではなく NHRP のオンデマンド解決**で
  実現すること（これが Phase 2 との違い）。

