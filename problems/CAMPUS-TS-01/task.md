# CAMPUS-TS-01: キャンパスLAN 障害切り分け演習

あなたは企業キャンパスネットワークの運用担当です。ユーザ部門から届いた
**障害チケット**（運用者から別途提示）をもとに原因を切り分け、修正してください。

## ネットワーク概要

```
                       [srv-net 10.20.0.0/24]
                        svr1(.10: DNS/DHCP/Web)
                             |
              inside(.1) [ asa1 ]           ← サーバ網はFW経由でアクセス
                             | outside 10.254.3.0/30
      core1 ============== core2            (iosv, OSPF area 0)
       |    \            /    |
       |     \          /     |             コア-ディストリ: /30 p2p
     dist1 ============ dist2               (SVI/HSRP/STP root/OSPF)
      /  \              /  \
    acc1  +------------+   acc2             (L2: VLAN/トランク/STP)
     |                     |   |
   cli10                cli30 cli40         (DHCP クライアント)
```

| VLAN | 用途 | サブネット | GW(HSRP VIP) |
|------|------|-----------|--------------|
| 10 | Users-A | 10.10.10.0/24 | 10.10.10.1 |
| 20 | Users-B | 10.10.20.0/24 | 10.10.20.1 |
| 30 | Guest | 10.10.30.0/24 | 10.10.30.1 |
| 40 | IoT | 10.10.40.0/24 | 10.10.40.1 |
| – | Server | 10.20.0.0/24 | 10.20.0.1 (asa1 inside) |

- ルーティング: OSPF area 0（コア・ディストリ・asa1）
- 冗長設計: VLAN10/30 は dist1 系、VLAN20/40 は dist2 系が正系
  （STP root と HSRP Active を一致させる設計）
- DHCP/DNS/Web は svr1（10.20.0.10）。各 SVI からリレー
- core1-dist 間はレガシー転送区間（MTU 1400）を通っている

## ログイン情報

- ネットワーク機器（IOS/ASA）: コンソールから `SUZUKI` / `CCNPccnp`
  （enable も同じ。★本ラボは ASA のパスワードポリシーに合わせ 8 文字）
- Linux（svr1 / cli10 / cli30 / cli40）: `suzuki` / `CCNP`

## ルール

- 障害は **1 件** 仕込まれています（設計と異なる設定が 1 箇所）
- 機器の設定変更で復旧してください。トポロジ変更・ケーブル差し替えは禁止
- svr1（Linux サーバ）は正常です。サーバ側の変更は不要
- 復旧後、チケットの症状が解消していることを確認すること

## 提出

原因（どの機器のどの設定か）・根拠にした show コマンド・修正内容を報告してください。
採点は運用者が `campus_ops.py grade` で行います（golden=100点）。
