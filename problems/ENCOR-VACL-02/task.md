# 問題 ENCOR-VACL-02 : VACL で特定ホスト間の通信を遮断

## シナリオ
スイッチ **SW01** の VLAN **10 (USERS, 192.168.10.0/24)** に、4 台のルータが
エンドポイントとして接続されています。

| ルータ | VLAN10 内 IP |
|--------|--------------|
| RT01 | 192.168.10.1 |
| RT02 | 192.168.10.2 |
| RT03 | 192.168.10.3 |
| RT04 | 192.168.10.4 |

セキュリティ要件により、**RT01 (192.168.10.1) と RT03 (192.168.10.3) の間の通信を
完全に遮断**したい。それ以外のホスト間の通信（RT01⇔RT02、RT02⇔RT03、RT01⇔RT04 など）は
これまでどおり通すこと。

## 到達目標 — SW01 のみ
- **192.168.10.1 と 192.168.10.3 の間の通信を遮断**する（双方向）。
- VLAN 10 内のそれ以外の通信は転送する（止めない）。

## 制約
- 設定するのは **SW01 のみ**。RT01〜RT04 は変更しない。
- ルータ（L3）側ではなく、**スイッチ側で VLAN 内のトラフィックに作用させる**こと。

## テスト方法（自己確認用）
各ルータには SSH でログインできます。設定後、ルータから相互に
`ping` を打って効果を確認できます。
- RT01 (10.1.10.12) から `ping 192.168.10.3` → 失敗するはず
- RT01 から `ping 192.168.10.2` / RT02 から `ping 192.168.10.3` → 成功するはず

## アクセス
- SW01: `10.1.10.11`（Telnet, SUZUKI / CCNP）
- RT01: `10.1.10.12` / RT02: `10.1.10.13` / RT03: `10.1.10.14` / RT04: `10.1.10.15`（SSH, SUZUKI / CCNP）

## 採点
```
ansible-playbook playbooks/grade.yml -e problem=ENCOR-VACL-02 --vault-password-file <(printf 'CCNP\n')
```
