# ENARSI-URPF-01: エッジルータに送信元検証 (anti-spoofing) を導入せよ

難易度: ★★★★☆（4/6） / 配点: 100点 / 対象: ENARSI (300-410) インフラセキュリティ

## シナリオ

あなたは企業エッジルータ **RT01** の管理者。SOC の監査で
「インターネット境界で**送信元アドレス検証 (anti-spoofing)** が未実装」と指摘を受けた。

RT01 は 2 つの ISP にマルチホーム接続しており、経路は OSPF で受信している。

```
            RT01 (あなたの管理対象, Lo0 = 1.1.1.1)
       E0/0 |              | E0/1
    (Uplink-A)          (Uplink-B)
            |              |
      RT02 (ISP-A) ---- RT03 (ISP-B)
                (ISP間ピアリング)
```

- ISP 側には顧客網（192.168.100.0/24, 192.168.201.0/24 など）が収容されている。
- RT02 / RT03 は **ISP の設備**（設定変更は禁止。コンソールでの状態確認は可）。

## 要件

1. RT01 の**両アップリンク（E0/0, E0/1）の受信方向**で送信元アドレス検証
   （uRPF）を有効化せよ。
2. 検証モードは、セキュリティポリシー上 **各インタフェースで技術的に可能な限り
   厳格なもの**を使用すること。
3. ただし、以下の**正規フロー**は維持しなければならない
   （断は重大障害としてカウントされ大きく減点）:
   - ISP-A NOC からの死活監視: RT02 発 `ping 1.1.1.1 source 2.2.2.2`
   - 顧客B からの通信: RT03 発 `ping 1.1.1.1 source 192.168.100.1`
4. RT01 の**ルーティング設定（OSPF・静的経路）の変更・追加は禁止**
   （経路設計は ISP との契約で固定されている）。
5. RT02 / RT03 の設定変更は禁止。

## 補足

- 採点は設定の字面ではなく**状態と実際のドロップ挙動**を見る。
- 作業前に、要件 3 の正規フローが現状すべて成功することを確認しておくとよい。
