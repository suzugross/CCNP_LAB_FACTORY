# 問題 : サイト間 IPsec VPN の構築（ルーテッドVPN方式）

## シナリオ
本社 (RT01) と支店 (RT02) が、通信事業者の WAN 網 (RT04) を介して接続されている。
情報システム部が作成した下記の**設定仕様書**に従い、本社–支店間にサイト間
**IPsec VPN** を構築し、拠点 LAN 同士を動的ルーティングで相互接続すること。

- RT01 = **本社 (HQ)** / RT02 = **支店 (Branch)**
- RT04 = **WAN トランジット**（事業者網・**変更禁止**）

## 既設構成（変更しないこと）
- **Underlay (WAN)**: 各拠点は RT04 へ /30 で接続済み。他拠点へは RT04 経由
  （デフォルトルート設定済み）。RT01=`10.0.14.1/30` / RT02=`10.0.24.1/30`。
- **Loopback0**: RT01=`1.1.1.1/32` / RT02=`2.2.2.2/32`。
- **Loopback10（拠点LAN相当）**: RT01=`192.168.1.1/24` / RT02=`192.168.2.1/24`。

## 設定仕様書（この値のとおり実装すること）

### 1. トンネル方式
| 項目 | 指定値 |
|------|--------|
| 方式 | ルーテッドVPN（トンネルIFにIPsecを直接載せる）。**GRE は使用しない** |
| トンネルIF | 両拠点とも **`Tunnel0`** |
| トンネルアドレス | **`10.255.12.0/30`**（本社 `.1` / 支店 `.2`） |
| トンネル source | 各拠点の WAN 側物理IF |
| IF の IPv4 MTU | **1400** バイト |
| TCP MSS 書き換え | **1360** バイト |

### 2. IKE（フェーズ1）
| 項目 | 指定値 |
|------|--------|
| バージョン | **IKEv1** |
| 認証方式 | 事前共有鍵 **`Ss2026#WanHqB1`**（対向グローバルアドレス限定で定義） |
| 暗号化 | **AES-256** |
| ハッシュ | **SHA-256** |
| DH グループ | **14** |
| SA ライフタイム | 既定値のまま |
| デッドピア検出 | **間隔 10 秒 / リトライ 3 秒 / 常時送信（periodic）** |

### 3. IPsec（フェーズ2）
| 項目 | 指定値 |
|------|--------|
| ESP 暗号化 | **AES-256** |
| ESP 整合性 | **HMAC-SHA-256** |
| カプセル化モード | **tunnel** |
| PFS | **DH グループ 14** |
| SA ライフタイム | 既定値のまま |

### 4. ルーティング
| 項目 | 指定値 |
|------|--------|
| プロトコル | **OSPF**（プロセスID 任意）**エリア 0** |
| 広告対象 | トンネル区間・各拠点 LAN (`192.168.x.0/24`) |
| router-id | 各ルータの Loopback0 アドレス |
| 経路要件 | 対向拠点の LAN を **/24 のプレフィックス長のまま**学習すること |

## 到達目標
- 両拠点間で IPsec セッションが確立し、仕様書のパラメータが実機の状態として
  確認できること。
- 拠点 LAN 間 (`192.168.1.1` ⇔ `192.168.2.1`) の通信が VPN 経由で成立すること。

## 制約
- 既設の underlay・アドレッシング・RT04 は変更しない。
- プロファイル名・トランスフォームセット名などの**オブジェクト名は任意**
  （仕様書に無い名前は自由に決めてよい）。
- スタティックルートの追加は禁止（既設のデフォルトルートのみ）。
