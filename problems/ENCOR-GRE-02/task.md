# 問題 ENCOR-GRE-02 : デュアル GRE トンネルで2拠点を収容 (overlay OSPF)

## シナリオ
本社 **HQ (RT01)** は、プロバイダ網 **RT02（変更禁止）** を経由して
2つの拠点 **Branch1 (RT03)** / **Branch2 (RT04)** に接続しています。
各サイトにはプライベート LAN（Loopback10 で擬似）があり、これらを
**プロバイダ網には一切載せず**、サイト間のオーバーレイ上だけで相互に到達させたい。

HQ から各 Branch へ **個別の GRE トンネル**を張り、オーバーレイの
ルーティングで各プライベート LAN を交換するよう構成してください。

## トポロジ
```
   RT03(Branch1)            RT04(Branch2)
        \                    /
   10.0.23.0/30        10.0.24.0/30
          \                /
           RT02 (ISP / transit・変更禁止)
                  |
            10.0.12.0/30
                  |
           RT01 (HQ / Hub)
```

- underlay は **OSPF プロセス 1 / area 0** で既設（各ルータの Loopback0 到達性は確保済み）。
- 各サイトのプライベート LAN（Loopback10）は **underlay に広告されていない**。

## アドレス台帳
| ルータ | Loopback0 | Private LAN (Lo10) | transit 直結 |
|--------|-----------|--------------------|--------------|
| RT01 (HQ)      | 1.1.1.1/32 | 10.10.1.1/32 | 10.0.12.1 |
| RT02 (transit) | 2.2.2.2/32 | —            | — |
| RT03 (Branch1) | 3.3.3.3/32 | 10.10.3.1/32 | 10.0.23.2 |
| RT04 (Branch2) | 4.4.4.4/32 | 10.10.4.1/32 | 10.0.24.2 |

## 到達目標
1. **HQ–Branch1** を `Tunnel100`、**HQ–Branch2** を `Tunnel200` の GRE トンネルで接続する。
   - トンネルのオーバーレイ・アドレス: `Tunnel100` = `172.16.13.0/30`（HQ=.1 / Branch1=.2）、
     `Tunnel200` = `172.16.14.0/30`（HQ=.1 / Branch2=.2）。
2. オーバーレイのルーティングは **OSPF プロセス 2 / area 0** で構成し、
   各サイトのプライベート LAN（Lo10）と各トンネルを参加させる。
3. 結果として、HQ は両 Branch のプライベート LAN を、各 Branch は HQ の
   プライベート LAN を、**トンネル経由で**学習している状態にすること。

## 制約
- 設定するのは **RT01 / RT03 / RT04**。**RT02 は一切変更しない**。
- プライベート LAN を underlay（OSPF プロセス 1）に載せない。
- 物理リンク断・帯域変更などの手段は不可。
