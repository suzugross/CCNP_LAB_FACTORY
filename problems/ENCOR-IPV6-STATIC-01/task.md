# 問題 ENCOR-IPV6-STATIC-01 : IPv6 アドレッシング + 静的ルーティング（難易度4）

## シナリオ
**IPv6 のみ**で運用する小規模ネットワーク。RT01—RT02—RT03 のチェーン構成。
動的ルーティングは使わず、**静的経路だけ**で全 Loopback の相互到達を作る。
初期状態はデータリンクが `up` なだけで、**IPv6 アドレス・`ipv6 unicast-routing`・Loopback・
静的経路はすべて未設定**。これらを一から構成する。

## トポロジ
```
       (Lo0)                      (Lo0)                      (Lo0)
   2001:DB8:A::1/128         2001:DB8:B::2/128         2001:DB8:C::3/128
        RT01 ─── 2001:DB8:12::/64 ─── RT02 ─── 2001:DB8:23::/64 ─── RT03
            Et0/0              Et0/0    Et0/1              Et0/0
```

## アドレッシング仕様
各ルータに以下を設定する。**`ipv6 unicast-routing` を必ず有効化**すること。

| ルータ | Loopback0 (GUA) | リンク GUA | リンクローカル (手動) |
|--------|------------------|------------|------------------------|
| RT01 | `2001:DB8:A::1/128` | Et0/0 = `2001:DB8:12::1/64` | Et0/0 = `FE80::1` |
| RT02 | `2001:DB8:B::2/128` | Et0/0 = `2001:DB8:12::2/64`<br>Et0/1 = `2001:DB8:23::2/64` | Et0/0 = `FE80::2`<br>Et0/1 = `FE80::2` |
| RT03 | `2001:DB8:C::3/128` | Et0/0 = `2001:DB8:23::3/64` | Et0/0 = `FE80::3` |

- リンクローカルアドレスは**自動生成に任せず、上表の値を手動で**設定すること
  （静的経路がこの値を参照するため）。
- RT02 は両リンクで同じ `FE80::2` を使う（リンクローカルは**リンクごと**に一意であれば良い）。

## 到達目標（静的経路のみ）
全ルータが他の全 Loopback に到達できるようにする。ただし**経路の作り方を指定**する
（IPv6 静的経路の 3 つの書き方を使い分けること）:

1. **RT01 → RT02 の Loopback (`2001:DB8:B::2`)**:
   **再帰静的経路**（ネクストホップに対向の**グローバルアドレス**を指定。出口IFは書かない）。
2. **RT01 → RT03 の Loopback (`2001:DB8:C::3`)**:
   ネクストホップに**リンクローカルアドレス**を使った静的経路。
3. **RT02 → RT01 の Loopback (`2001:DB8:A::1`)**:
   ネクストホップに**リンクローカルアドレス**を使った静的経路。
4. **RT02 → RT03 の Loopback (`2001:DB8:C::3`)**:
   **再帰静的経路**（対向のグローバルアドレス）。
5. **RT03**: 個別経路ではなく **IPv6 デフォルト経路 (`::/0`)** を 1 本だけ書き、
   RT01・RT02 の Loopback の両方に到達できるようにする（ネクストホップは**リンクローカル**）。

## 制約・ヒント（最小限）
- 動的ルーティングプロトコル（OSPFv3/EIGRP/BGP 等）は使わない。**静的経路のみ**。
- リンクローカルアドレスは複数リンクで重複し得るため、**そのまま使うと困る場面がある**。
  IOS のエラー／警告をよく読むこと。
- 戻りの経路も忘れずに。ping は往復で成立する。

## アクセス・採点
SSH `SUZUKI / CCNP`（RT01=.11 / RT02=.12 / RT03=.13）。
```
ansible-playbook playbooks/grade.yml -e problem=ENCOR-IPV6-STATIC-01 --vault-password-file <(printf 'CCNP\n')
```
※ 採点は (1) 手動リンクローカルの設定、(2) 各静的経路の**方式**（再帰／出口IF+リンクローカル／
デフォルト経路）を `running-config` で確認し、(3) **実 ping** で全 Loopback 相互到達を能動確認します。
