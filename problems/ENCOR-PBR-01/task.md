# 問題 ENCOR-PBR-01 : 送信元別の経路振り分け

## シナリオ
支社ルータ **RT01** は 2 つの社内セグメントを収容している。

- **営業部** … `Loopback1` = `10.1.1.1`
- **開発部** … `Loopback2` = `10.2.2.1`

両セグメントはデータセンター上のサーバ **`3.3.3.3`** (RT03 の Loopback0) と通信する。
RT01 から DC へは RT02 経由・RT03 直結の 2 経路があり、OSPF はすでに全台で稼働して
全到達できている。

## トポロジ
```
            10.0.12.0/30
      RT01 ───────────── RT02
       │  e0/0      e0/0   │ e0/1
  e0/1 │                   │ 10.0.23.0/30
       │ 10.0.13.0/30      │
       └──────── RT03 ─────┘
          e0/0       e0/1
```
| ルータ | Loopback | 直結 |
|--------|----------|------|
| RT01 | Lo0 1.1.1.1/32 ／ Lo1 10.1.1.1/32(営業) ／ Lo2 10.2.2.1/32(開発) | RT02=10.0.12.2 ／ RT03=10.0.13.2 |
| RT02 | Lo0 2.2.2.2/32 | RT01=10.0.12.1 ／ RT03=10.0.23.2 |
| RT03 | Lo0 3.3.3.3/32 (DC サーバ) | RT01=10.0.13.1 ／ RT02=10.0.23.1 |

- OSPF (プロセス 1 / area 0) は 3 台とも設定済み・全到達可能（初期状態で疎通あり）。

## 到達目標 — RT01 のみ
1. **開発部 (`10.2.2.1`) を送信元**とする DC (`3.3.3.3`) 宛の通信は **RT02 (`10.0.12.2`) 経由**で
   届くようにする。
2. **営業部 (`10.1.1.1`) を送信元**とする DC (`3.3.3.3`) 宛の通信は **現状のまま（最短経路）** とする。
3. どちらのセグメントからも `3.3.3.3` への到達性は維持すること。

## 制約
- 設定を変更してよいのは **RT01 のみ**。RT02 / RT03 は変更しない。
- スタティックルートで OSPF の最短路自体を書き換えてはならない（経路表の最短路は維持）。

## アクセス（SSH, SUZUKI / CCNP）
- RT01: `10.1.10.11` ／ RT02: `10.1.10.12` ／ RT03: `10.1.10.13`
- ※CML コンソールから直接設定して解いて構いません。
