# 問題 : IKEv2 サイト間 IPsec VPN の構築（本社＋2支店）

## シナリオ
本社 (RT01) と 2 つの支店 (RT02 / RT03) が、通信事業者の WAN 網 (RT04) に収容されている。
セキュリティ部門が策定した下記の**設定仕様書**（次世代暗号 = NGE 準拠）に従い、
本社をハブとする **IKEv2 のサイト間 IPsec VPN** を 2 本構築し、全拠点の LAN を
動的ルーティングで相互接続すること。**支店間の通信は本社経由**で成立させる。

- RT01 = **本社 (HQ)** / RT02 = **支店1** / RT03 = **支店2**
- RT04 = **WAN トランジット**（事業者網・**変更禁止**）

## 既設構成（変更しないこと）
- **Underlay (WAN)**: 各拠点は RT04 へ /30 で接続済み。他拠点へは RT04 経由
  （デフォルトルート設定済み）。RT01=`10.0.14.1/30` / RT02=`10.0.24.1/30` /
  RT03=`10.0.34.1/30`。
- **Loopback0**: RT01=`1.1.1.1/32` / RT02=`2.2.2.2/32` / RT03=`3.3.3.3/32`。
- **Loopback10（拠点LAN相当）**: RT01=`192.168.1.1/24` / RT02=`192.168.2.1/24` /
  RT03=`192.168.3.1/24`。

## 設定仕様書（この値のとおり実装すること）

### 1. トンネル方式（2本の point-to-point ルーテッドVPN）
| 項目 | 指定値 |
|------|--------|
| 方式 | ルーテッドVPN（トンネルIFにIPsecを直接載せる）。**GRE は使用しない** |
| 本社–支店1 | 本社 **`Tunnel1`** ⇔ 支店1 **`Tunnel1`** / **`10.255.12.0/30`**（本社 `.1` / 支店1 `.2`） |
| 本社–支店2 | 本社 **`Tunnel2`** ⇔ 支店2 **`Tunnel1`** / **`10.255.13.0/30`**（本社 `.1` / 支店2 `.2`） |
| トンネル source | 各拠点の WAN 側物理IF |
| IF の IPv4 MTU | **1400** バイト |
| TCP MSS 書き換え | **1360** バイト |

### 2. IKE（IKE SA）
| 項目 | 指定値 |
|------|--------|
| バージョン | **IKEv2**（スマートデフォルトに頼らず明示定義すること） |
| 暗号化 | **AES-GCM-256** |
| PRF | **SHA-384** |
| DH グループ | **19** |
| 認証方式 | 事前共有鍵（**双方向**とも） |
| 事前共有鍵 | 本社–支店1: **`Ss2026#HqB1-v2`** / 本社–支店2: **`Ss2026#HqB2-v2`**（対向アドレス単位で定義） |
| SA ライフタイム | 既定値のまま |
| デッドピア検出 | **間隔 10 秒 / リトライ 3 秒 / on-demand 方式** |

### 3. IPsec（Child SA）
| 項目 | 指定値 |
|------|--------|
| ESP 暗号化 | **AES-GCM-256** |
| ESP 整合性 | 追加指定なし（暗号化方式の特性を踏まえて構成すること） |
| カプセル化モード | **tunnel** |
| PFS | **DH グループ 19** |
| SA ライフタイム | 既定値のまま |

### 4. ルーティング
| 項目 | 指定値 |
|------|--------|
| プロトコル | **EIGRP AS 100** |
| 広告対象 | 両トンネル区間・各拠点 LAN (`192.168.x.0/24`) |
| router-id | 各ルータの Loopback0 アドレス |
| 経路要件 | 支店1 ⇔ 支店2 の LAN 経路が**本社経由**で成立すること |

## 到達目標
- 本社で 2 本の IKEv2 セッションが確立し、仕様書のパラメータ（暗号スイート含む）が
  実機の状態として確認できること。
- 全拠点 LAN 間の通信が VPN 経由で成立すること（支店間 `192.168.2.1` ⇔ `192.168.3.1`
  は本社経由）。

## 制約
- 既設の underlay・アドレッシング・RT04 は変更しない。
- プロポーザル名・キーリング名・プロファイル名などの**オブジェクト名は任意**。
- スタティックルートの追加は禁止（既設のデフォルトルートのみ）。
- 支店間を直接結ぶトンネルは作らない（トンネルは仕様書の 2 本のみ）。
