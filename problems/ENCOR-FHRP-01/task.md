# 問題 ENCOR-FHRP-01 : HSRP による冗長ゲートウェイ

## シナリオ
クライアント VLAN10（192.168.10.0/24）の既定ゲートウェイを **HSRP** で冗長化します。
L2 スイッチ（SW01/SW02）と VLAN10・トランクは構成済み。ルータ RT01/RT02 の
VLAN10 側インタフェースには IP も設定済みです。あなたは **HSRP の設定**を行います。

## 構成（設定済み）
```
   RT01 (.2) ── SW01 ═══trunk═══ SW02 ── RT02 (.3)
                     VLAN10 (192.168.10.0/24)
```
| 機器 | VLAN10 側 IP |
|------|--------------|
| RT01 | 192.168.10.2 |
| RT02 | 192.168.10.3 |
| 仮想IP (HSRP) | **192.168.10.1** ← これを構成する |

## 🎯 タスク
RT01・RT02 の VLAN10 側インタフェースで **HSRP グループ 10** を構成する：
1. 仮想IP（ゲートウェイ）= **192.168.10.1**
2. **RT01 を Active**（priority 110・preempt 有効）
3. RT02 は既定（priority 100）で **Standby**
4. HSRP **version 2** を使用

## 制約
- スイッチ側のL2構成（VLAN/トランク）は変更しないこと。

> 採点: `ansible-playbook playbooks/grade.yml -e problem=ENCOR-FHRP-01`
