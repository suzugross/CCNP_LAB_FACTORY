# 問題 ENARSI-BGP-NHSELF-01 : iBGP next-hop-self で外部経路を AS 内に届ける

## シナリオ
あなたの AS (**AS 65001**) は、境界ルータ **RT01** で外部 **RT03 (AS 65003)** と
eBGP 接続しています。RT03 は `203.0.113.0/24` を広告しています。
AS 内には **RT02** があり、RT01 と **iBGP** で接続済みです。

RT01 で BGP を構成して RT03 から経路を受け取り、AS 内の RT02 にも
**実際に使える形で**届けてください。

## トポロジ
```
RT03 (AS65003) ──eBGP── RT01 (AS65001) ──iBGP── RT02 (AS65001)
        10.1.13.0/30              10.1.12.0/30
```
| ルータ | AS | Loopback0 | RT01 との直結 |
|--------|----|-----------|----------------|
| RT01 | 65001 | 1.1.1.1/32 | — |
| RT02 | 65001 | 2.2.2.2/32 | 10.1.12.2 (iBGP) |
| RT03 | 65003 | 3.3.3.3/32 | 10.1.13.2 (eBGP) |

- RT01–RT02: `10.1.12.0/30`（RT01=.1 / RT02=.2）AS65001 内部
- RT01–RT03: `10.1.13.0/30`（RT01=.1 / RT03=.2）外部
- RT02 は外部リンク (`10.1.13.0/30`) への経路を持っていません。
- RT02 は RT01 と iBGP 構築済み（変更不可）。RT03 は eBGP 構築済み（変更不可）。

## 到達目標 — RT01 のみ
1. RT03 と eBGP、RT02 と iBGP をそれぞれ確立する。
2. **RT02 が `203.0.113.0/24` を BGP 経路として実際に RIB に持つ**ようにする。
   - 単に iBGP で広告が届くだけでは RT02 の RIB には入りません。RT02 が
     その経路を**実際に使える next-hop**で学習する状態にしてください。

## 制約
- 設定するのは **RT01 のみ**。RT02 / RT03 は変更しない。
- IGP の追加や外部リンクの再配送で「next-hop を別途到達可能にする」のではなく、
  **iBGP の next-hop の扱いそのもの**で解決すること。

## アクセス（SSH, SUZUKI / CCNP）
- RT01: `10.1.10.11` / RT02: `10.1.10.12` / RT03: `10.1.10.13`
