# 問題 ENARSI-DMVPN-BGP-01 : DMVPN 支店網とデータセンタを相互接続する

## シナリオ
2 つの支店 (RT02 / RT03) を本社ハブ (RT01) に **DMVPN** で収容し、さらにハブの
背後にあるデータセンタ (DC = RT05) と支店を相互に到達させたい。DC 側は **BGP** で
DC LAN を広告しており、支店側は **EIGRP** オーバーレイで動いている。ハブが両者の
**境界**となり、ルーティング情報を相互に橋渡しする。

- RT01 = **ハブ (NHS / 再配送境界)** … 受験者が構築
- RT02 = **スポーク1** / RT03 = **スポーク2** … 受験者が構築
- RT04 = **WAN トランジット** (NBMA 中継・**変更禁止**)
- RT05 = **DC コア** (eBGP・DC LAN 広告・**変更禁止**)

## 既設構成（変更しないこと）
- **Underlay (NBMA)**: 各エッジは RT04 へ /30 接続、他拠点 NBMA へは RT04 経由
  （デフォルトルート設定済み）。RT01=`10.0.14.1` / RT02=`10.0.24.1` / RT03=`10.0.34.1`。
- **ハブ⇔DC**: `10.0.15.0/30`（RT01=`.1` / RT05=`.2`）。物理 IP は設定済み。
- **Loopback0**: RT01=`1.1.1.1` / RT02=`2.2.2.2` / RT03=`3.3.3.3`。
- **DC (RT05, 変更禁止)**: AS `65000`、`172.20.20.0/24` を BGP で広告。ハブと eBGP を張る。

## 到達目標
1. **DMVPN (Phase 2)**: 1 本の `Tunnel0` でマルチポイントオーバーレイを確立。両スポークが
   ハブに登録され UP。EIGRP AS100 で全拠点が互いの Loopback0 を学習。
   **支店間 (RT02⇔RT03) はハブを経由せず直接**通信する（対向スポーク経路の
   ネクストホップが**ハブでなく対向スポーク自身**であること）。
2. **境界 (ハブ) でのルーティング相互橋渡し**:
   - ハブは DC (RT05, AS65000) と **eBGP** を確立する（ハブ AS = `65001`）。
   - **支店 → DC 方向**: 両スポークが DC LAN `172.20.20.0/24` を学習し、DC へ到達できる。
   - **DC → 支店 方向**: DC (RT05) が両スポークの Loopback0 (`2.2.2.2`/`3.3.3.3`) を
     学習し、支店へ到達できる。

## 指定値（採点が固定値で判定するため必ず従うこと）
- トンネル: **`Tunnel0`** / オーバーレイ **`10.255.0.0/24`**（Hub`.1`/Sp1`.2`/Sp2`.3`）
- NHRP **network-id `1`** / **認証キー `DMVPNKEY`**
- オーバーレイ IGP: **EIGRP AS `100`** / ハブの NBMA = `10.0.14.1`
- ハブの BGP AS = **`65001`** / DC の AS = `65000`（peer = `10.0.15.2`）

## 制約
- 既設の underlay / アドレッシング / RT04 / RT05 は変更しない。
- 暗号化 (IPsec) は不要。トンネルは `Tunnel0` 1 本のみ。
- ルーティングの相互橋渡しは**ハブ 1 か所**で行うこと。

## 採点
```
ansible-playbook playbooks/grade.yml -e problem=ENARSI-DMVPN-BGP-01 --vault-password-file <(printf 'CCNP\n')
```
> IOSv のため採点は CML コンソール経由（`access: console`）。
> 支店間ダイレクト通信・支店⇔DC 疎通は採点時に能動 ping で確認する。
