# 問題 ENCOR-COPP-02 : CoPP 多層防御 (ICMP制限 + Telnet遮断)

## シナリオ
RT01 の制御プレーン保護を強化することになった。社内は SSH に統一されており、
ルータ自身宛の Telnet (TCP/23) は攻撃のみと判断できる。一方、運用上 ICMP は
完全に止めたくないが、過剰な流量で CPU を圧迫されるのは避けたい。

## 到達目標
RT01 の**制御プレーン入力**に対し、以下の効果を**同一のポリシー**で実現すること。

1. **ICMP**: **8000 bps** に制限する。**適合は通常転送・超過は破棄**。
2. **Telnet (TCP 宛先ポート 23)**: 制御プレーンに到達したものを**すべて破棄**する
   （適合・超過の区別なく落とす）。
3. **その他の制御プレーン宛トラフィック** (OSPF / SSH / BGP など) には影響を与えないこと。

## 制約
- 既存のインタフェース／到達性の設定は変更しないこと。
- ICMP と Telnet で**それぞれ独立した分類**を用いること（1 つのクラスに混在させない）。
- クラス名・ポリシー名・ACL 名は任意。
