# 問題 ENARSI-BGP-ORIGIN-01 : BGP origin コードでベストパスを選択

## シナリオ
RT01 (AS 65001) は、上流 **RT02 (AS 65002)** と **RT03 (AS 65003)** に eBGP 接続します。
両者は同一の anycast 宛先 **`203.0.113.0/24`** を広告しています。
運用ポリシーにより、この宛先へのトラフィックは **RT02 経由を優先**したい。

ところが両経路は **weight・local-preference・AS-path 長・MED がすべて同条件**で、
通常のタイブレークでは安定して RT02 が選ばれません。
**より上位のタイブレーク要素**を使って、RT02 経由を確定的に勝たせてください。

## 構成（初期状態で投入済み）
| ルータ | Loopback0 | RT01 との直結 |
|--------|-----------|----------------|
| RT01 (AS65001) | 1.1.1.1/32 | — |
| RT02 (AS65002) | 2.2.2.2/32 | 10.1.12.2 |
| RT03 (AS65003) | 3.3.3.3/32 | 10.1.13.2 |

- RT01–RT02: `10.1.12.0/30`（RT01=.1 / RT02=.2）
- RT01–RT03: `10.1.13.0/30`（RT01=.1 / RT03=.2）
- RT02 / RT03 はともに `203.0.113.0/24` を広告済み（変更不可）。

## 到達目標 — RT01 のみ
1. RT02・RT03 と eBGP を確立する。
2. `203.0.113.0/24` のベストパスが **RT02 経由 (next-hop 10.1.12.2)** になるようにする。

## 制約
- 設定するのは **RT01 のみ**。RT02 / RT03 は変更しない。
- **weight / local-preference / MED / AS-path prepend は使用しないこと。**
  これらより上位で効くタイブレーク要素で制御すること。
- スタティック・デフォルトルートで「見せかけ」の到達性を作らない。

## アクセス（SSH, SUZUKI / CCNP）
- RT01: `10.1.10.11` / RT02: `10.1.10.12` / RT03: `10.1.10.13`
