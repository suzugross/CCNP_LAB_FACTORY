# ENCOR-QOS-CLASS-01: 信頼境界を作れ — 入口での分類とマーキング（効果実測つき）

難易度: ★★★☆☆（3/6） / 配点: 100点 / 対象: ENCOR (350-401) QoS

QoS 体感シリーズ第1問。学習順序: **CLASS-01 → POLICE-01 → LLQ-01**。

## シナリオ

2拠点ネットワークの QoS 導入プロジェクトが始まった。最初の仕事は
**信頼境界（trust boundary）をネットワークの入口に作る**こと。

拠点1 の端末はマーキングをせず（あるいは勝手な値を付けて）送ってくる。
そこで **RT01 の LAN 側入口**で、社内の QoS 設計に従ってトラフィックを
分類・マーキングすることになった。

```
PC01 ── RT01 ──[ WAN ]── RT02 ── PC02
拠点1          10.1.12.0/30       拠点2
 .10   E0/0(入口)    E0/1   E0/1   E0/0   .20
192.168.10.0/24              192.168.20.0/24
```

- PC01 / PC02: Ubuntu（ログイン `SUZUKI` / `CCNP`）。iperf3 導入済み。
  PC02 では iperf3 サーバが常駐起動している。
- RT01 / RT02 の IP・経路は設定済み（**変更禁止**）。
- **RT02 には到着パケットの DSCP を数える観測ポリシー `MONITOR` が
  仕込んである**（変更禁止）。マーキングが WAN の向こうまで届いたかは
  `show policy-map interface Ethernet0/1 input`（RT02）で確認できる。

## 社内 QoS 設計（マーキング仕様）

| 社内システム | トラフィックの実体 | マーキング |
|--------------|--------------------|-----------|
| 音声ストリーム | UDP 宛先ポート 5201 | **EF** |
| バックアップ転送 | TCP 宛先ポート 5201 | **AF11** |
| 死活監視 | ICMP | **CS2** |

## 要件

1. RT01 の**LAN 側入口**で、上表の 3 種類のトラフィックを分類し、
   指定の DSCP をマーキングせよ。
2. 上表にないトラフィックのマーキングは変更しなくてよい。
3. 設定変更は **RT01 のみ**（RT02・PC の変更は禁止。既存のアドレス・経路も変更禁止）。
4. class-map / policy-map / ACL 等の**名前は任意**（採点は名前でなく状態と効果を見る）。

## 体感手順 — マーキングが「本当に飛んでいる」ことを数値で確かめる

PC01 から 3 種類のトラフィックを流す:

```
# 音声ストリーム模擬 (UDP 5201)
iperf3 -c 192.168.20.20 -u -b 500K -l 200 -t 5

# バックアップ転送模擬 (TCP 5201)
iperf3 -c 192.168.20.20 -t 5

# 死活監視模擬 (ICMP)
ping -c 10 192.168.20.20
```

観測ポイントは2つ:

1. **RT01（マーキングした側）**: `show policy-map interface Ethernet0/0`
   — 各クラスの packets / Packets marked が「意図したクラスだけ」増えるか。
2. **RT02（受け取った側）**: `show policy-map interface Ethernet0/1 input`
   — OBS-EF / OBS-AF11 / OBS-CS2 のカウンタが増えていれば、
   マーキングが WAN を越えて伝搬した証拠。

**記録表**（実装前後で RT02 の MONITOR カウンタを記録）:

| RT02 観測クラス | 実装前 packets | 実装後 packets |
|-----------------|----------------|----------------|
| OBS-EF (音声) | | |
| OBS-AF11 (バックアップ) | | |
| OBS-CS2 (監視) | | |

「なぜ実装前は全部 0 なのか」「マーキングはどの機器のどの向きで行われ、
どこまで保持されるのか」を自分の言葉で説明できたら完成。

## 採点

- 採点は自動（構成の状態 + 効果の実測）。採点器も PC01 から 3 種のトラフィックを流し、
  RT01 のマーキング実績と RT02 の到着カウンタを確認する。
- 目安時間: 30〜45 分。
