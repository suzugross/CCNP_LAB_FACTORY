# 問題 ENCOR-EIGRP-BUILD-01 : named-mode EIGRP を要件どおり構築する（難易度4）

## シナリオ
コア3台（RT01/RT02/RT03）とスポーク2台（RT04/RT05）から成る企業網です。各リンクと
Loopback には IP 設定済みですが、**動的ルーティングは未構成**です。下記の要件をすべて満たす
**named mode の EIGRP** を構成し、全拠点の到達性を成立させてください。

```
        RT01
       /    \         コア3角（RT01–RT03 は広域・高遅延リンク）
     RT02 — RT03
      |       |
     RT04    RT05     スポーク
```

## 構成（初期状態で投入済み・変更不可）
| リンク | セグメント | 備考 |
|--------|-----------|------|
| RT01–RT02 | 10.1.12.0/30 | RT01=.1 / RT02=.2 |
| RT01–RT03 | 10.1.13.0/30 | RT01=.1 / RT03=.2　**広域・高遅延** |
| RT02–RT03 | 10.1.23.0/30 | RT02=.1 / RT03=.2 |
| RT02–RT04 | 10.2.24.0/30 | RT02=.1 / RT04=.2 |
| RT03–RT05 | 10.3.35.0/30 | RT03=.1 / RT05=.2 |

| ルータ | Loopback0(RID用) | 業務 Loopback |
|--------|------------------|----------------|
| RT01 | 10.0.1.1/32 | — |
| RT02 | 10.0.2.2/32 | — |
| RT03 | 10.0.3.3/32 | — |
| RT04 | 10.0.4.4/32 | 10.4.0.1, 10.4.1.1, 10.4.2.1（各/32） |
| RT05 | 10.0.5.5/32 | 10.5.0.1, 10.5.1.1, 10.5.2.1（各/32）＋ **管理 10.99.5.1/32** |

## 要件（採点項目）
1. **named mode で構成**：`router eigrp <名前>` ＋ `address-family ipv4 unicast autonomous-system 100`。
   **AS は 100**。各ルータで **eigrp router-id を手動設定**（Loopback0 のアドレス）。
2. **有効化インタフェースを厳密に**：参加させるのは各 L3 リンクと Loopback のみ。
   `network 0.0.0.0` のような全許可は使わず、必要な範囲だけを `network` で指定すること。
   **Loopback は passive-interface**（隣接を張らない）、L3 リンクのみ隣接を形成すること。
3. **認証**：**RT01–RT02 リンクのみ** に EIGRP 認証（MD5）を設定。鍵は両端一致させ、
   **他のリンクには認証を付けない**こと（誤って全 IF に付けると他隣接が壊れる）。
4. **不等コストロードバランシング**：RT01 から **10.5.0.0/22**（RT05 業務ブロックの集約）への
   経路が **2 つの next-hop（RT02 経由と RT03 直行）でロードバランス**されるよう構成すること。
   ※ RT01–RT03 が高遅延のため既定では 1 経路のみ。Feasible Successor を活かす設定が必要。
5. **スポーク最適化**：RT04・RT05 を **EIGRP スタブ（connected + summary）** として構成すること。
6. **集約**：コアに対し、RT02 は **10.4.0.0/22**、RT03 は **10.5.0.0/22** の集約 1 本のみを広告すること
   （構成要素の /32 を個別に広告しない）。
7. **フィルタ**：RT05 の管理 Loopback `10.99.5.1/32` は、**RT03 からは到達可能（管理用に残す）**
   としつつ、**RT01・RT02 へは伝播させない**こと。
   ※ この /32 は集約ブロック外（集約では抑止できない）。かつ RT03 には見せる必要があるため
   「EIGRP に載せない」では要件を満たせず、**中継点 RT03 でのフィルタ**が必要。

## 到達目標（最終状態）
- 全ルータが相互の業務ネットワーク／集約へ到達。
- RT01 の `10.5.0.0/22` が **2 next-hop** でインストールされている。
- コアのルーティングテーブルに `10.4.0.0/22` `10.5.0.0/22` が **集約 1 本** で現れ、
  構成要素 /32 は **現れない**。
- 管理 `10.99.5.1/32` は **RT03 の RIB には在り（到達可）**、**RT01・RT02 の RIB には無い**こと。

## 制約
- スタティックルートは使用しないこと。EIGRP プロセスは **named mode** であること（classic 不可）。
